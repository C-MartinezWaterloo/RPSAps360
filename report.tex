\begin{abstract}
This report summarizes an end-to-end pipeline for predicting \texttt{TransactionPrice} from housing transaction data, compares a classic hedonic regression baseline to multiple neural tabular models (ANN, FM, DeepFM, TabNet), and strengthens evaluation via multiple random seeds, multiple data splits, and a time-based split (training on earlier years and testing on later years). On the full feature set with a 70/15/15 random split, the best ANN achieves test RMSE(log) \(\approx\)\BestAnnTestRmse with \(\approx\)\BestAnnTestAcc\% accuracy, compared to a hedonic baseline at \BestHedTestRmse RMSE(log) and \BestHedTestAcc\% accuracy. Results are tracked in a single canonical spreadsheet (\texttt{results\_all.csv}); an interactive dashboard (\texttt{plots/report\_graphs.html}) provides one-click PNG/SVG exports of all plots.
\end{abstract}

\section{Introduction (problem, data source, and motivation)}
\paragraph{Problem statement.}
Given a housing transaction record with property attributes (e.g., size, beds/baths), location signals (e.g., FSA/city, lat/lon), and transaction time (year/quarter), the goal is to predict \texttt{TransactionPrice}. We treat this as a regression task and train all models to predict \(\log(1+\texttt{TransactionPrice})\) for numerical stability.

\subsection{Project proposal context (data source and motivation)}
The project proposal describes the RPS housing database as an aggregation of multiple sources (including appraisal workflows and franchise back-office transaction feeds). In particular, RPS coordinates on the order of \(\sim\)\num{180000} appraisals/year and extracts structured information from standardized appraisal forms. Appraisal reports include neighborhood/site descriptions, building attributes (exterior/interior), and a sales-comparison approach with comparable recent transactions.   \item \textbf{Comparable-sales emphasis:} the proposal emphasizes the sales comparison approach (three or more recent comparable MLS sales in geographic/time proximity) as a key driver of appraisal value, suggesting strong nonlinear and interaction effects between location, time, and property attributes.

\end{itemize}

\section{Data preparation (cleaning and tensorization)}
All modeling in this repository starts from a cleaned CSV (\texttt{cleaned\_transaction\_gtha-2.csv}, kept local) and converts it into a single PyTorch tensor file using \texttt{prepare\_ann\_tensors.py}. The preparation step performs:
\begin{itemize}[leftmargin=*]
  \item \textbf{Target filtering:} drops rows with missing/non-positive \texttt{TransactionPrice}.
  \item \textbf{Missing-value handling:} treats tokens such as \texttt{NA}, \texttt{NULL}, and sentinel values (e.g.\ \texttt{-999}) as missing.
  \item \textbf{Numeric features:} computes streaming mean/std (Welford) and standardizes; missing numeric values are imputed with the mean.
  \item \textbf{Categorical features:} normalizes strings (uppercasing), maps missing to \texttt{\_\_NA\_\_}, and builds integer vocabularies for embedding-style models (optional cap with \texttt{\_\_OTHER\_\_}).
  \item \textbf{Targets:} stores both raw \texttt{y} and \texttt{y\_log1p = log1p(y)} for stable training.
\end{itemize}

\subsection{Two-pass tensorization (implementation details)}
\begin{itemize}[leftmargin=*]
  \item \textbf{Pass 1 (streaming scan):} iterate through the CSV once to (i) compute mean/std for each numeric feature using Welford’s algorithm, (ii) count category frequencies for each categorical column, and (iii) count how many rows are kept after target filtering.
  \item \textbf{Vocabulary build:} sort categories by frequency and map them to integer indices; reserve \texttt{\_\_NA\_\_} for missing values and optionally \texttt{\_\_OTHER\_\_} for rare categories if a cap is enabled.
  \item \textbf{Pass 2 (tensor materialization):} allocate arrays of the final size and re-scan the CSV to populate standardized numeric tensors and categorical index tensors. Numeric missing values are imputed with the training-wide mean; categorical missing values map to \texttt{\_\_NA\_\_}.
  \item \textbf{Saved metadata:} numeric/categorical column lists, numeric stats, category maps, and the chosen \texttt{feature\_set}. This keeps training scripts simple and makes experiments reproducible.
\end{itemize}

\section{Models}
\begin{itemize}[leftmargin=*]
  \item \textbf{Hedonic regression (linear fixed effects):} linear model on standardized numeric features plus categorical offsets (implemented as 1D embeddings). This matches the classic hedonic pricing framing where price is decomposed into contributions from attributes and location/time effects \cite{rosen1974hedonic}.
  \item \textbf{ANN (embeddings + MLP):} categorical embeddings concatenated with numeric features and passed through an MLP.
  \item \textbf{FM:} Factorization Machine capturing pairwise interactions via low-rank factors \cite{rendle2010fm}.
  \item \textbf{DeepFM:} linear + FM interactions + a deep MLP \cite{guo2017deepfm}.
  \item \textbf{TabNet:} sequential feature selection with sparse masks \cite{arik2019tabnet}.
  \item \textbf{Planned baselines: XGBoost/LightGBM.} These are often very strong for tabular data \cite{chen2016xgboost,ke2017lightgbm} but require compiled dependencies. A runner script is provided for external execution: \texttt{models/06\_xgboost/train\_boosting.py}.
\end{itemize}

\section{Related work (why these baselines)}
\paragraph{Hedonic regression.}
The ``hedonic'' baseline is a long-standing approach in housing economics and appraisal practice, where value is modeled as a function of observable attributes and location/time effects \cite{rosen1974hedonic}. In modern ML terms, it is a strong linear baseline on well-engineered features and a useful reference point for whether nonlinear/interaction models add measurable value.

\paragraph{Neural tabular models and interactions.}
Tabular prediction often benefits from modeling interactions between variables (e.g., size$\times$location, time$\times$neighborhood). Factorization Machines (FM) efficiently represent pairwise interactions through low-rank factors \cite{rendle2010fm}, while DeepFM combines linear terms, FM interactions, and a deep network \cite{guo2017deepfm}. TabNet provides an alternative inductive bias via attentive feature selection and sparse masks \cite{arik2019tabnet}.

\paragraph{Tree boosting as a standard comparator.}
Gradient-boosted decision trees (XGBoost/LightGBM) are widely used as strong baselines for tabular tasks \cite{chen2016xgboost,ke2017lightgbm}. This repo keeps boosting runs optional (external environment) but includes a runner script for fair comparison.

\section{Experimental setup and metrics}
\subsection{Splits}
\begin{itemize}[leftmargin=*]
  \item \textbf{Random split:} standard 70/15/15 random partition (used for most sweeps).
  \item \textbf{Time split:} sort by (\texttt{TransactionYear}, \texttt{Quarter}) and split 70/15/15 by time order to evaluate future generalization.
\end{itemize}

\subsection{Metrics}
All models predict \texttt{log1p(TransactionPrice)} and report:
\begin{itemize}[leftmargin=*]
  \item \textbf{RMSE(log):} root mean squared error in log1p space (RMSLE-like; lower is better).
  \item \textbf{MdAPE:} median absolute percentage error in price space (robust to outliers).
  \item \textbf{Accuracy\%:} derived as \(\max(0, 100\cdot(1-\text{MdAPE}))\).
\end{itemize}

\paragraph{Formal definitions.}
Let \(p_i\) be the true price and let the model predict \(\hat{y}_i \approx \log(1+p_i)\).
Define \(y_i = \log(1+p_i)\) and \(\hat{p}_i = \exp(\hat{y}_i)-1\).
Then:
\[
  \mathrm{RMSE}_{\log} = \sqrt{\frac{1}{N}\sum_{i=1}^N(\hat{y}_i - y_i)^2},
\]
\[
  \mathrm{APE}_i = \frac{|\hat{p}_i - p_i|}{p_i},\quad
  \mathrm{MdAPE} = \mathrm{median}_i(\mathrm{APE}_i),\quad
  \mathrm{Accuracy}\% = \max\!\bigl(0,\,100\cdot(1-\mathrm{MdAPE})\bigr).
\]

\subsection{Checkpoint selection, overfitting, and test-set hygiene}
Neural models are trained for a fixed number of epochs but select the \emph{best validation checkpoint} (lowest validation MSE) for final reporting. For many sweeps, we also log \texttt{*\_last} metrics (final epoch) so the report can illustrate overfitting (e.g., validation performance that degrades after the best epoch).

To keep evaluation clean:
\begin{itemize}[leftmargin=*]
  \item \textbf{Test-clean sweeps:} \texttt{sweep\_ann.py} selects hyperparameters using validation only and evaluates the test set once per selected configuration.
  \item \textbf{Exploratory sweeps:} \texttt{sweep\_ann\_deep\_test.py} computes test metrics for every run; these rows are useful for exploration but should not be used for final model selection.
\end{itemize}

\paragraph{Why do some runs use \texttt{train\_max\_samples=50,000}?}
Large randomized sweeps often cap training rows for speed (especially for TabNet/DeepFM sweeps). The \emph{best} headline results in this report include full-train runs (\(\approx\)\NTrain training rows) explicitly labeled in \texttt{results\_all.csv}.

\subsection{Robustness}
To ensure observed differences are not due to a favorable split or initialization, the best ANN configuration and the hedonic baseline are re-run across multiple \texttt{split\_seed}s and \texttt{seed}s using \texttt{eval\_suite.py}. These runs are recorded under \texttt{source=eval\_suite} in \texttt{results\_all.csv}.

\section{Hyperparameter and architecture conclusions (so far)}
This section summarizes repeatable patterns observed in \texttt{results\_all.csv} and \texttt{REPORT.md}. It is intended as ``grab bag'' language for a final write-up.

\subsection{ANN (embeddings + MLP)}
\begin{itemize}[leftmargin=*]
  \item \textbf{Depth helps on full features.} The best full-train run uses a deeper MLP (\texttt{512→256→128→64}) rather than a shallow network.
  \item \textbf{Regularization matters.} Moderate dropout (\(\approx\)0.05--0.10) and weight decay (\(\approx\)\(5\times10^{-4}\) to \(10^{-3}\)) improve validation/test performance and reduce overfitting.
  \item \textbf{Learning rate sweet spot.} For the best full-feature runs, \texttt{lr} around \(\approx\)0.003--0.004 performs best; overly small or large learning rates degrade performance.
  \item \textbf{Embeddings: diminishing returns.} Increasing \texttt{embed\_dim\_cap} above 64 did not consistently improve the top runs in this project.
\end{itemize}

\subsection{FM and DeepFM}
\begin{itemize}[leftmargin=*]
  \item \textbf{FM improves over linear on capped-data sweeps,} but tends to have worse tail error than the best ANN/DeepFM full-train runs.
  \item \textbf{DeepFM benefits from both interactions and depth.} The best DeepFM configurations combine a nontrivial factor dimension (\texttt{k}) with a large MLP; however, these sweeps were often run with a 50k train cap for speed.
\end{itemize}

\subsection{TabNet}
Across the initial sweep, TabNet underperformed ANN/DeepFM in this setup. This may be due to (i) limited tuning, (ii) sensitivity to optimization settings, and/or (iii) the strong benefit of learned embeddings for high-cardinality categoricals.

\section{Discussion}
\paragraph{Why the ANN outperforms hedonic regression.}
Both models benefit strongly from adding time and location signals (e.g.\ \texttt{TransactionYear}, \texttt{FSA}, \texttt{gcode\_City}, lat/lon). The ANN improves further by learning nonlinear effects and interactions (e.g.\ diminishing returns to size, location-dependent trends, and cross-feature interactions) that a linear hedonic model cannot represent.

\paragraph{Random vs time splits.}
Random splits measure interpolation across mixed time periods; time splits are closer to a forecasting evaluation. Performance drops notably under time splits for all models, but the ANN retains a large advantage (\(\approx\) 13.4 percentage points higher test accuracy than the hedonic baseline on average in our time-split checks).

\section{Ethical considerations and responsible use}
\begin{itemize}[leftmargin=*]
  \item \textbf{Privacy and data governance.} This report intentionally avoids address-level identifiers and uses aggregated geographic signals (e.g., FSA/city, lat/lon) to reduce privacy risk. Any real deployment should confirm legal/contractual permission for model training, retention policies, and strict access controls to the underlying dataset.
  \item \textbf{Bias and downstream impact.} Housing prices reflect historical inequities; a predictive model can encode and amplify these patterns. If used in lending, insurance, tenant screening, or neighborhood ``risk'' scoring, this can create harmful feedback loops. Responsible use should include subgroup performance analysis (by region, property type, time window), careful policy constraints, and human oversight.
  \item \textbf{Transparency.} We include interpretable baselines (hedonic regression), feature-importance analysis, and time-split evaluation. For high-stakes use, we recommend documenting model limitations, monitoring drift, and providing clear guidance on when predictions should \emph{not} be trusted (e.g., out-of-distribution transactions).
\end{itemize}

\section{Limitations and next steps}
\begin{itemize}[leftmargin=*]
  \item \textbf{Preprocessing leakage:} current tensorization computes numeric mean/std and categorical vocab using the full dataset. A stricter setup would compute these on training data only per split.
  \item \textbf{Sweep hygiene:} \texttt{sweep\_ann\_deep\_test.py} evaluates test every run and should be treated as exploratory; \texttt{sweep\_ann.py} is the test-clean sweep.
  \item \textbf{Never-before-seen final test:} reserve a strict holdout window (preferably the latest year/quarter block) that is not touched by any sweeps or early stopping decisions, and evaluate once after locking the model and hyperparameters. If feasible, obtain a fresh extract after the current snapshot cut-off (true new data).
  \item \textbf{Boosting baselines:} run XGBoost/LightGBM externally using \texttt{models/06\_xgboost/train\_boosting.py}, then merge \texttt{xgb\_runs.csv}/\texttt{lgbm\_runs.csv} into \texttt{results\_all.csv} via \texttt{export\_results.py}.
  \item \textbf{Interpretability:} extend feature analysis (e.g.\ partial dependence-style perturbations) on top drivers such as \texttt{LivingArea} and \texttt{TransactionYear}.
\end{itemize}

\section{Conclusion}
On a large cleaned housing transaction dataset (\NTotal rows), a neural tabular ANN with categorical embeddings and a moderately deep MLP substantially outperforms a hedonic linear fixed-effects baseline on a random split (\BestAnnTestRmse vs \BestHedTestRmse test RMSE(log)). Robustness checks across multiple seeds and splits confirm this advantage is consistent, and time-based evaluation shows expected degradation while preserving a clear margin over the baseline. Overall, the strongest gains come from richer feature sets (especially location + time) and from models that can capture nonlinearities and interactions.

\section{Appendix D: Report-ready interpretation snippets}
\paragraph{High-level (non-technical) wording.}
Using a strong set of property, location, and time features, the neural network model predicts transaction prices substantially more accurately than a traditional hedonic regression baseline. The largest improvements come from capturing nonlinear relationships (e.g., diminishing returns to size) and interactions between location and time (market shifts that differ by neighborhood).

\paragraph{Technical wording.}
On the full feature set, the ANN achieves materially lower log-space error than the hedonic baseline and maintains this advantage under multiple random seeds and split seeds. Under time-based evaluation (train earlier years, test later years), all models degrade, but the ANN generalizes significantly better, indicating that its gains are not solely an artifact of random mixing across time.
